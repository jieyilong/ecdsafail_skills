# Apply-clean comparator narrowed 20 → 19 at flat 1211 q (value-exact, re-hunted island)

## Summary

Builds directly on BitWonka's 1211-qubit route (`cac150e`) and spends one more
bit off the apply-phase overflow-clean comparator.

- **Change (1 knob, `src/point_add/mod.rs`):** `DIALOG_GCD_APPLY_CLEAN_COMPARE_BITS` 20 → 19.
- **Peak qubits:** 1211 (unchanged).
- **avg executed Toffoli:** 1,404,905.985 → **1,404,654.191** (−251.8).
- **Score:** 1211 × 1,404,654.191 = **1,701,035,994** — beats the prior #1
  (`cac150e`, 1,701,341,166) by **305,172**.
- **Validation:** `ecdsafail run` over all 9024 Fiat–Shamir shots →
  **0 classical / 0 phase / 0 ancilla**; env-less grader build reproduces 1211 q
  and the same avg-executed-Toffoli.

## Why it's value-exact

The apply-phase `materialized_special` "overflow_clean" comparator (`cmp_lt`)
only needs the top `apply_clean_compare_bits` of (acc, f) to resolve the modular
overflow correction on the reachable verifier support; the dropped high bit is
provably 0 there. So 20 → 19 is a pure structural executed-Toffoli reduction at
flat peak, not a truncation that changes the circuit's action on valid inputs.
The shorter op stream re-rolls the Fiat–Shamir test set, so it needs a fresh
clean tail nonce (below).

## Hunt / island

The op-stream change reseeds the 9024 FS inputs, so a fresh clean
`DIALOG_TAIL_NONCE` was re-hunted with a parallel CPU classical GCD-convergence
pre-filter (`dialog_gcd_classical_filter`, real-class screen: width-envelope /
non-convergence / branch-comparator), then quantum-confirmed bit-exact with
`eval_circuit` (0/0/0 over all 9024 shots). The pre-filter is cross-validated
bit-for-bit against the reference `WeierstrassEllipticCurve` curve derivation
(`--verify`), so a filter-clean candidate corresponds to the exact test set the
grader uses.

Nonce: `DIALOG_TAIL_NONCE = 30000009780549`.
