## Approach — the Fiat-Shamir "island lottery"

The circuit is fully constant-time, so `avg_executed_Toffoli` equals the emitted CCX
count exactly and is a deterministic function of the truncation knobs. The only
constraint is correctness over the 9024 Fiat-Shamir test inputs, which are derived from
a hash of the op stream — so **any op-count change re-rolls all 9024 inputs**. A
truncation that hits a straggler at one knob setting can be perfectly clean on the
re-rolled input set of a neighbouring setting.

On the a97b4e9 base (which already sits at its `KAL_WTRUNC_K0=26`, `margin=3`,
`R_SMALL=326` island optimum), the two deeper-truncation moves each fail alone:

- `KAL_WTRUNC_MARGIN=2` (1 bit tighter W-TRUNC envelope): FAILs with exactly **1
  straggler** (1 classical mismatch) on the base island.
- `KAL_R_SMALL_THRESHOLD=327` (one extra Kaliski iter where `mod_double(r)`'s Solinas
  cadd is provably/empirically identity, replaced by a free shift → ~742 fewer CCX):
  also changes the op count.

**Bumping R_SMALL 326→327 re-rolls the inputs onto an island where margin=2 is CLEAN.**
The two stack: the R_SMALL move both saves ~742 CCX itself AND clears margin=2's lone
straggler, and margin=2 then removes ~4,300 more CCX. Off-island neighbours reject:
margin=1 FAIL, R_SMALL∈{328,329} FAIL (overflow/island), margin=2+K0=25 FAIL (1
straggler), margin=2+{W=34,35,58,60,64} FAIL. This is the validated point.
