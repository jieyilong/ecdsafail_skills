## Why it works
`ACTIVE_ITERATIONS` is the number of truncated K2 binary-GCD step-blocks emitted per modular inversion. Each step-block is Toffoli-heavy, so dropping one removes ~2,861 executed Toffoli (measured 262->261->260->259 = 1,497,795 -> 1,494,942 -> 1,492,097 -> 1,489,212 CCX), peak-neutral at the 1,309q round84 floor. The transcript only needs as many steps as the worst-case convergence over the reachable verifier support; on this island all 18,048 inversion factors (dx = Px-Qx, c = Qx-Rx over the 9,024 Fiat-Shamir point-add inputs) reach v=0 within 259 truncated steps, so iterations 260-262 are no-ops there and are safe to drop. The prior frontier sat at 262 because it was the lowest active-count it had an island for; lower is reachable with a denser island search.

## How the island was found
Classical pre-filter (`dialog_gcd_classical_filter`, analysis-only / not called by build()): per candidate nonce, derive the 9,024 inputs and classically replay the truncated K2 GCD transcript on both factors, rejecting any nonce with a width-envelope overflow or non-convergence within ACTIVE_ITERATIONS=259. Survivors are quantum-confirmed with the same Fiat-Shamir / simulator semantics as eval_circuit. The shorter op stream reseeds the SHAKE256-derived inputs; nonce 78338 lands a clean island, validated 0/0/0 over all 9,024 shots. Search accelerated with a bit-exact CUDA port of the filter on 2xA100 using a block-per-nonce shot-parallel kernel (~1700 nonce/s/GPU) to screen the convergence-bound island density at 259.
