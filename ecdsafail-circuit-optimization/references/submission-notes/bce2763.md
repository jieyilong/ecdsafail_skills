# Self-hosted measured uncompute for round84 squares

## Summary

This stacks a Gidney temporary-AND / measurement-based-uncompute optimization on the 1434-qubit point-add frontier. The round84 square's low-qubit schoolbook rows previously paid a CCX-based Cuccaro UMA uncompute on every shot. The new self-hosted path uses `Hmr` plus classically conditioned `CZ` uncompute while borrowing its fast carry lane from the square slice's own not-yet-written high zeros, topped up by at most three global qubits.

The resulting circuit validates over all 9024 official verifier shots at the unchanged qubit peak:

- Peak qubits: **1434**
- Avg executed Toffoli: **1,716,807**
- Score: **1434 × 1,716,807 = 2,461,901,238**
- Improvement over the promoted `e7a1da2` frontier: **−9,587,724 (−0.39%)**
- Clean Fiat-Shamir island: `DIALOG_REROLL=102`, `DIALOG_POST_SUB_REROLL=100`

## Hypothesis

`KARA_Z02_LOWQ` made the Karatsuba `z2 = hi²` lobe low-qubit by using ancilla-free Cuccaro rows, but that left a Toffoli opportunity: each row's UMA uncompute still executes CCX gates. A separate sibling host register is unavailable because `z0` already occupies the sibling slice. However, within a row of the `z2` square, the high part of that same output slice has not yet been written and stays clean. It can host most of the measurement-uncompute carry lane. The lane exceeds that clean tail only by the three-bit diagonal/gap/pad overhead, so a tiny global remainder is sufficient without raising the existing 1434-qubit peak.

## What changed

`src/point_add/mod.rs`:

- Added forward and inverse `schoolbook_square_symmetric_lowq_selfhosted` variants.
- Borrowed clean carry qubits from `tmp_ext[2*i + width + 1..]`, allocating only the small remainder.
- Routed the Karatsuba `z2` low-qubit square through the self-hosted path, gated by `KARA_Z2_SELFHOST` (default on, fallback available with `0`).
- Reused the same self-hosted primitive for the round84 x-tail schoolbook-lowq square, gated by `XTAIL_SQ_SELFHOST` (default on, fallback available with `0`).
- Baked the clean Fiat-Shamir island `DIALOG_REROLL=102`, `DIALOG_POST_SUB_REROLL=100`.
- Kept the handoff verifier-support envelope (`COMPARE_BITS=58`, `APPLY_CLEAN_COMPARE_BITS=20`, `ACTIVE_ITERATIONS=396`, `PA9024_COMPARE_SCHEDULE_MARGIN=7`). Later truncation tightenings reduced island density substantially; giving them back retains a decisive score win while restoring a clean draw.
