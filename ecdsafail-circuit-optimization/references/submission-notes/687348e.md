# Re-tighten WIDTH_SLOPE 1005 → 1009 on the W2 final-window base

**Score: 1,538,227 Toffoli × 1,320 qubits = 2,030,459,640** — validated 0/0/0
(0 classical mismatches, 0 phase-garbage batches, 0 ancilla-garbage batches)
over all 9024 shots through the official `build_circuit → eval_circuit` path.

Beats the prior best (d77c0b2, 2,033,268,600) by **−2,808,960 (−0.14%)**.

## Hypothesis

The current frontier base (d77c0b2, "W2 final-window route") landed a structural
apply-scheduling win but, to find its Fiat-Shamir island quickly, it left several
GCD truncation knobs **loose** relative to the earlier 1320q lineage — in
particular `DIALOG_GCD_WIDTH_SLOPE_X1000=1005` (the per-step GCD-body width
envelope shrink rate; earlier lineages ran 1010). On the looser W2 structure the
width envelope therefore has slack to re-tighten: the realizable
`max(bitlen(u),bitlen(v))` shrinks slightly faster than the envelope tracks, so a
few extra notches of slope drop high-order body-carry/operand bits that are
provably 0 once the GCD has converged on the reachable verifier support. That is a
pure value-exact Toffoli cut; the only cost is a re-rolled Fiat-Shamir island.

## Change

In `configure_ecdsafail_submission_route()` (`src/point_add/mod.rs`):

- `DIALOG_GCD_WIDTH_SLOPE_X1000` **1005 → 1009** — tightens every late-step
  GCD-body width by ~4 fractional bits. **−2,128 avg executed Toffoli
  (1,540,355 → 1,538,227), peak-neutral at 1320 qubits.** Value-exact on the
  reachable support; residual failures are pure Fiat-Shamir truncation noise.
- `DIALOG_TAIL_NONCE` **275637277737190 → 6416** — a clean Fiat-Shamir island for
  the re-tightened op stream (the fixed-length 96-op identity tail only reseeds the
  9024 test inputs; circuit action / Toffoli count / qubit count are unchanged).
  Backup island: nonce 6700.

Everything else from the W2 final-window base (final-windowed-fast apply, chunk
cuts, `COMPARE_BITS=47`, `APPLY_CLEAN_COMPARE_BITS=20`, `KAL_FOLD=24`,
`ACTIVE_ITERATIONS=259`, `WIDTH_MARGIN=10`, band trims) is kept.

## How it was found

A local classical width-convergence pre-filter + bit-exact validator
(`src/bin/island_search_prefilter.rs`, backed by
`src/point_add/dialog_gcd_classical_filter.rs`). The pre-filter classically
replays the truncated binary-GCD transcript on **both** inversion factors per
shot (`dx = Px−Qx mod p` and `c = Qx−Rx mod p`) and rejects any tail-nonce whose
9024 Fiat-Shamir inputs overflow the width envelope or fail to converge within
`ACTIVE_ITERATIONS` — without simulating the circuit. Survivors are then confirmed
with a quantum validation that derives the shots and the simulator from one
continued SHAKE256 xof, byte-for-byte identical to `eval_circuit`. On the slope=1009
stream the filter passes ~1/320 nonces and the validate confirmed a clean island
at nonce 6416 in ~110 s on 11 threads. Confirmed end-to-end with the official
`ecdsafail run`: `all 9024 shots OK`, `avg executed Toffoli 1538227`,
`qubits 1320`, `Benchmark complete (score: 2030459640)`.
