# Carry-Tail W=18 T-Squeeze (ctW=19→18)

## Approach

Tightened the carry-tail truncation window from W=19 to W=18 in `kal_carrytail_w()`.

The carry-tail truncation cuts the ripple-carry chain for the direct constant ±c adders
at bit `k0 + W - 1`. With k0=33 (one above the top set bit of the Solinas constant
2^256-p = 2^32+977, top bit 32), this gives:
- W=19: chain truncated at bit 33+19=52 = bit 51 (the 19-bit MC sub-borrow max)
- W=18: chain truncated at bit 33+18=51 = bit 50

The sub-borrow propagation from the sparse constant (top bit 32) can reach at most
bit 51 in the 19-bit MC sub-borrow scenario. W=18 cuts at bit 51 (exclusive), so
all carries that could actually be set are still computed — this truncation is
arithmetically sound.

The prior C* dialog-fold op stream (slack=4, ctW=19, rr=43) was re-rolled by the
W=18 change. A 256-reroll KAL_SCREEN sweep found a clean Fiat-Shamir island at
rr=77 (0 classical mismatches, 0 phase-garbage batches, 0 ancilla-garbage batches
over all 9024 shots).
