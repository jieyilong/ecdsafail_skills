## What changed

`src/point_add/cuccaro.rs` + `src/point_add/kaliski_state.rs` only.

The base already applies a "MAJ-fold" to the **SUB**-path constant adder
(`csub_nbit_const_direct_fast`, gated by `KAL_MAJFOLD_SUB`): it rewrites the 3-CCX
borrow majority `maj(!acc_i, ctrl, borrow_in)` into **1 CCX + free CX** using the
borrow-in as pivot, via the identity `maj(a,b,d) = d ^ ((a^d) & (b^d))`. The computed
value is identical, so the backward measurement-uncompute (Hmr + `cz_if`) is unchanged.

This submission mirrors that exact technique onto the **ADD**-path constant adder
`cadd_nbit_const_direct_fast`, whose forward carry sweep had the identical untouched
3-CCX majority `carry_{i+1} = maj(acc_i, k_i, carry_i)`:

```
b.ccx(acc[i], ci, target);      //  3 CCX  ->  fold to:
b.ccx(ctrl,  acc[i], target);   //    cx(ci, acc[i]); cx(ci, ctrl);
b.ccx(ctrl,  ci, target);       //    ccx(acc[i], ctrl, target);
                                //    cx(ci, target); cx(ci, ctrl); cx(ci, acc[i]);
```

i.e. **−2 CCX per carry bit** on every constant-ADD in the Kaliski / modular-reduction
path, gated by a new `majfold_add_enabled()` flag (env `KAL_MAJFOLD_ADD`, default ON).
Net: −5,184 avg-executed Toffoli per shot, peak-neutral.

Only the **forward** carry sweep yields Toffoli savings — the ADD reverse/uncompute
sweep is a measurement-uncompute (Hmr + `cz_if`, Clifford), so there is no Toffoli to
fold there.

## What did NOT work (recorded for others)

Enabling carry-tail **truncation on the ADD path** (mirroring the SUB-path carry-tail)
is a structural dead-end: it produces ~141 phase-garbage batches at every truncation
width and on the MAJ-fold-re-rolled island, so it is island-invariant, not a tuning
miss. Root cause: the ADD reverse sweep uses the *complemented* post-sum identity
`maj(!acc_i_final, k_i, carry_i)` (an `X` wraps the `cz_if` triplet), which
X-conjugates the CZ triplet and couples truncation error into a relative phase. The
SUB reverse sweep uses the *un-complemented* form and stays truncation-clean. Opening
the ADD carry-tail would require a full reverse-sweep redesign.
