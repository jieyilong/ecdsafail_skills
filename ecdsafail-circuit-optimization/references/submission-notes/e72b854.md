# W-trunc K0 21→20 stacked on R_SMALL=327

## Approach

Tightened the W-trunc full-width prefix length `kal_wtrunc_k0()` from 21 to 20,
stacked on top of the prior R_SMALL=327 win (2002-qubit base).

The Kaliski inversion's CCX-bearing per-iteration width loops (STEP-0 OR chain,
STEP-2 gt comparator, STEP-4 load/sub/transform/add) are sized by an empirical
bitlen envelope: full width `n` for the first `K0` iterations, then a linear decay
`n - floor((iter-K0)*2/3)`. Lowering K0 from 21 to 20 starts the envelope decay
one iteration earlier, narrowing the width loops in the mid iterations and shaving
**~4,400 avg-executed Toffoli** with zero change to peak qubits (2002).

## Why it now validates (it didn't before)

The repo history notes K0=20 had "no clean margin=0 island" on the older
R_SMALL=321 stream — every reroll failed. But the R_SMALL=327 change (prior win)
shifted the op count, which re-rolls the SHAKE256-derived Fiat-Shamir test inputs.
On the new R_SMALL=327 + K0=20 stream, a 0-127 KAL_SCREEN sweep found a clean
island at rr=13 (0 classical mismatches / 0 phase garbage / 0 ancilla garbage over
all 9024 shots). This is the standard "island lottery" co-tuning: the envelope
truncation is empirically sound, and the reroll knob (identity X;X pairs, zero
Toffoli/qubit cost) lands a clean test-input set.
