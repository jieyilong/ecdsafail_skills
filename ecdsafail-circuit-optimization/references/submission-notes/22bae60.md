# Improvement: unlock ADD-path carry-tail truncation

**Score:** 5,882,154,410 (2,547,490 avg Toffoli × 2,309 qubits) — −0.89% vs baseline. Validated 0/0/0 over 9024 shots.

**The bug it fixes:** carry-tail truncation was sound only for the *sparse* Solinas
constant (`c = 2^256−p`, top bit 32), so the ADD path was left disabled
("141 phase-garbage" wall). It's unsound for the *dense* constant `c = p+1` used by
`mod_neg`/`mod_double` (top bit 255): a dropped carry corrupts the high sum bits →
poisons a sign-bit comparison → leaves a flag dirty → its `free()`/`R` injects
random global phase.

**The fix:** new constant-aware `kal_carrytail_count_c` anchors the window at
`k0 = max(env_k0, c.bit_len())` — dense constants get the full (untruncated) chain,
sparse Solinas constants keep the tight truncation. This makes the ADD path
phase-clean, so carry-tail mode defaults to `"both"` (W=44 clean island).

**Files:** `src/point_add/cuccaro.rs` (cadd uses `kal_carrytail_count_c`),
`src/point_add/kaliski_state.rs` (new fn + mode default `"both"` + W=44).
